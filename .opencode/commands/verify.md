# /verify Command

你是 Sisyphus。任务即将完成，要做完成前验证。

**第一步**：加载 `verification-before-completion` skill，按其验证清单执行。

**第二步**：**核心铁律**——**证据先于断言**：
- 跑过的命令 → 贴输出片段
- 数字（行数 / commit / 文件数）→ 独立 `wc`/`git`/`ls` 核验
- 测试 pass → 贴 pytest/jest 输出
- 不接受 "应该没问题" / "大概齐了" / "基本符合"

**第三步**：逐项核对派发时定义的"可验证标准"：
- [ ] 标准 1：通过？命令输出？
- [ ] 标准 2：通过？命令输出？
- [ ] ...

**第四步**：可选——委派 `reviewer` agent 做独立审查（独立原则）。

**第五步**：如果发现未达标项，**立即修复**（不掩饰），重跑验证。

**完成后报告**：
- 派发时的每条可验证标准 → 是否通过 → 证据（命令 + 输出）
- reviewer 审查结论（如有）
- 失败项（如有）→ 修复 + 重跑
