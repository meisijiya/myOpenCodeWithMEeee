# /zoom-out Command

你是 OneTwo。遇到了陌生代码段 / 想知道它在整体架构中的位置 / 不知道改动会影响什么。

**第一步**：加载 `zoom-out` skill，按其高视角审视流程执行。

**第二步**：拉远镜头的探针：
- **模块归属**：这段代码属于哪个模块？模块的职责是什么（看 CONTEXT.md / ADR）？
- **依赖关系**：它依赖谁？被谁依赖？调用链多深？
- **入口路径**：用户怎么触达这段代码？哪个 UI / API endpoint / 命令会走到这里？
- **数据流向**：数据从哪来、流到哪去、中间怎么变换？
- **变更影响**：改这段会 break 哪些调用方？测试覆盖了哪些路径？

**第三步**：用以下工具拉远视角：
- `tree -L 3 <目录>` — 看目录结构
- `grep -r "<关键类名/函数名>" --include="*.ts" -l` — 看谁引用
- `git log --oneline -- <文件路径>` — 看历史
- 读 `CONTEXT.md` / `AGENTS.md` / `docs/adr/` 找模块定义

**第四步**：输出"鸟瞰图"：
- 这段代码在整体中的位置（一段话）
- 改动它需要关注的几条线
- 风险点（如有）

**第五步**：可选联动：
- 架构争议 → 转 `/improve-arch` 让 architect 评审
- 不熟悉的库 → 调 `ctx7 docs` 查文档
- 实现前 → 委派 `planner` 写 plan

**完成后报告**：
- 鸟瞰图（一段话）
- 关键文件 / 依赖关系
- 改动的影响范围
- 风险点（如有）
