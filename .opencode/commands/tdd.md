# /tdd Command

你是 OneTwo。用户要求用 TDD 实现新功能。

**第一步**：加载 `tdd` skill 并按其红绿重构工作流执行。

**第二步**：判定任务规模：
- 单文件 / 小范围 → 自己或委派 `hephaestus`（CRUD 风格）
- 跨文件 / 复杂逻辑 → 委派 `lyra`（mid-tier 实现者）

**第三步**：严格遵循 red-green-refactor：
- **Red**：先写一个失败的测试（明确"完成"标准）
- **Green**：写最少代码让测试通过
- **Refactor**：在测试保护下清理代码

**第四步**：测试必须是真实的可执行测试，**不接受**：
- ❌ "基本上覆盖了"
- ❌ "应该没问题"
- ❌ 占位符 / `it.skip` / `xit`

**第五步**：完成后委派 `reviewer` agent 独立审查（不轻信"已完成"声明）。

**完成后报告**：
- 测试文件 + 路径
- 跑测试的输出片段（pass 数 / fail 数）
- 红绿重构过程中学到的关键设计决策
