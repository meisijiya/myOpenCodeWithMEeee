# /prototype Command

你是 OneTwo。用户要试错设计 / 在动手前先看效果（不想污染主代码）。

**第一步**：加载 `prototype` skill，按其原型构建流程执行。

**第二步**：判断走哪条分支：
- **状态/业务逻辑问题** → 构建可运行终端 app（Python/Node 脚本），跑场景模拟
- **UI 设计问题** → 同一路由多个 UI 变体，可切换对比

**第三步**：原型铁律（**throwaway**）：
- ❌ 不写测试（原型不是产品）
- ❌ 不写文档（原型不是产品）
- ❌ 不优化性能（原型不是产品）
- ✅ 最小可跑通 + 验证假设
- ✅ 跑完即弃（不并入主代码）

**第四步**：原型完成后做"假设验证"：
- 假设 A → 跑原型 → 验证/证伪？
- 假设 B → 跑原型 → 验证/证伪？
- 设计决策 → 哪个变体最好？

**第五步**：原型走通后，**基于学到的经验**进入正式实现：
- 委派 `planner` 写 implementation plan
- 委派 `lyra` 实现
- 委派 `hephaestus` 写 boilerplate

**完成后报告**：
- 原型路径（一次性，跑完即删）
- 验证了/证伪了哪些假设
- 推荐的设计决策 + 理由
- 后续（写 plan / 实现 / 丢弃换方向）
