# /interview Command

你是 Sisyphus。需求不明确（缺 who / why / what / when / how / success / constraint），需要反问用户。

**第一步**：加载 `interview-me` skill 并按其工作流执行。

**第二步**：纪律（**单问题 + 高质量**）：
- 一次只问**一个**问题（不连珠炮）
- 每个问题都有明确意图（不是"你想要什么？"这种空问）
- 目标 ~95% 置信度后才停止

**第三步**：常用 5W 探针：
- **Who** — 谁会用？什么角色？技术背景？
- **Why** — 为什么现在需要？解决什么痛点？
- **What** — 输入输出？边界？edge case？
- **When** — 时间约束？依赖？里程碑？
- **How** — 性能 / 安全 / 可维护性约束？
- **Success** — "完成"长什么样？可观察的事实？
- **Out-of-scope** — 哪些明确不做？

**第四步**：可选联动：
- 答完澄清后 → `/plan` 命令进入计划阶段
- 涉及架构 → 委派 `architect` 评审
- 简单明确 → 自己动手

**完成后报告**：
- 关键澄清点（5W 答案）
- 置信度评估（~95%?）
- 下一步动作
